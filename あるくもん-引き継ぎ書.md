# あるくもん 引き継ぎ書（開発用）🔧

次の開発セッション（AI・人間どちらでも）がすぐ作業を再開できるようにするための技術文書。v1時点。

## 1. プロジェクト概要

- 歩数手入力式のモンスター育成×対戦ゲーム。ダイエットのモチベーション化が目的
- **単一HTMLファイル**（arukumon.html）完結。GitHub Pagesに `index.html` として配置して運用
- 外部依存はGoogle Fonts（DotGothic16）のみ。オフラインでもフォールバックで動作
- セーブは `localStorage`（キー: `arukumon_save_v1`）。使えない環境ではメモリ内フォールバック

## 2. ファイル内の構成（arukumon.html）

上から順に:
1. `<style>` — デザイントークン: シェル紺 #23253a / クリーム #fdf6e3 / 黄 #ffd447 / 属性色 P#ff5964・S#38a3ff・T#4cbb6c
2. `<body>` — 液晶(#lcd + canvas 160×112) / 5画面(home, battle, box, code, dex) / 下部ナビ / 歩数入力dialog
3. `const SPR` — ドット絵データ（16×16、`.`=透過、palで文字→色）
4. `/* ===LOGIC_START=== */ 〜 ===LOGIC_END=== */` — **純粋ロジック（DOM非依存）**。テストはこの区間を抽出して実行する
5. それ以降 — セーブ、描画、画面制御、バトルUI、ボックス、交配、図鑑

## 3. データ構造

```js
// モンスター
mon = { species, stage(0-3), total(累計歩数), entries:[{d:"YYYY-MM-DD", steps}](最大60件),
        hybrid, gen2, line("dog"|"rock"|"dragon"|"bird"), parentLines, seed }
// セーブ
G = { v:1, active:mon, freezer:[mon|null ×3], dex:{speciesId:true}, gym:0〜8, goal:目標歩数 }
```
- 日付キーは `todayKey(now?)`。**朝5時切り替わり**（`DAY_ROLLOVER_HOUR=5`、現在時刻−5時間の日付を使用）。0:00〜4:59の入力は前日あつかい

## 4. ゲームルール（実装値）

### 目標歩数システム（v1.1・てくてくエンジェル方式）
- 初回起動時にユーザーが **1日の目標歩数 `G.goal`**（**500**〜30000）を設定。図鑑画面の「1日の目標歩数を変更」でいつでも変更可
- 全しきい値・分岐・ステータスは目標基準。`DEFAULT_GOAL=7000`（goal未設定時のフォールバック）
- `childAt(g)=g×4` / `adultAt(g)=g×14` → **毎日達成なら14日で最終進化、達成率50%なら約28日**
- 途中で目標を変えるとしきい値も変わる（仕様として許容）

### 進化
- 幼年期化: 初回入力 ≥ 目標×0.8 → petiwan / 未満 → mokumo
- 成長期: 累計 ≥ childAt(goal) / 成熟期: 累計 ≥ adultAt(goal)
- **進化は1回の入力につき1段階まで**（addStepsのelse-if連鎖）。大幅超過達成でも段飛ばしせず、翌日の入力で次段階へ
- 分岐は進化時点の **直近7回平均の達成率 r = avg ÷ goal**

| 系統 | 成長期（r） | 成熟期の分岐（r） |
|---|---|---|
| dog | kowan（petiwan・r≥0.75） | ≥1.1 garuwolf / ≥0.6 pomerin / 未満 hesoten / **当日ゾロ目(111以上) daikichimaru（隠し・最優先判定）** |
| rock | goron（petiwan・r<0.75） | ≥1.0 metalgolem / kokedaman |
| dragon | picodora（mokumo・r≥0.75） | ≥1.0 burndorago / leafdora |
| bird | haneko（mokumo・r<0.75） | ≥1.0 sonicfalco / fukurou |

※達成率ちょうど100%継続の犬系はpomerin。garuwolfは目標を1割超える必要あり（テストで担保済み）

### 交配（hybrid）
- 双方 **成熟期のみ**。ボックスに空き1枠必須。卵は空き枠に入る
- ひかるたまごは孵化時に `parentLines` から系統を決定（seed+初回歩数で決定的）
- hybrid個体は成長期まで系統固定、成熟期で交配限定種へ:
  dog: 平均≥6000 dorawolf / angepome ・ rock: magmatitan ・ dragon: prismdorago ・ bird: phantomgriffon
- **両親とも交配限定種（gen2）→ daikichiking**

### ステータス・バトル
```
eff = total ÷ goal   // 「目標◯日ぶん」換算。強さは歩数の絶対量でなく努力量で決まる
HP  = [20,60,90,130][stage] + (hybrid成熟+15) + min(120, floor(eff×4))   // 上限: 目標30日ぶん
ATK = [4,10,16,24][stage]  + (hybrid成熟+4)  + min(40,  floor(eff×1.4)) // 上限: 目標約29日ぶん
SPD = (種族タイプS?12:8) + min(10, floor(eff÷3))
ダメージ = floor((技威力 + ATK×0.6) × 相性 × 0.9〜1.1)
相性: P>S>T>P、有利1.5 / 不利0.75（※現状維持で確定済み）
→ 目標3000歩×14日達成 と 目標7000歩×14日達成 は同ステータス（テストで担保済み）
```
- 行動順はSPD、同値はプレイヤー先手。CPUは40%で有利技を狙い、残りランダム
- ジムは `GYM` 配列8人（累計3.5万〜32万相当）。勝利で `G.gym++`
- 参戦条件: stage≥2。冷凍中の個体も選出可（`pickFighter`）

### コード仕様
```
"ARK1." + checksum(2桁base36) + "." + btoa(encodeURIComponent(JSON))
payload: {v:1, s:種族, t:累計, h:hybrid, g:gen2, l:系統, tr:トレーナー名, gl:目標歩数}
```
チェックサム不一致・未知種族・v違いは `decodeMon` が null を返す（改ざん耐性は簡易）。相手のステータスは相手側の `gl` で計算されるため、目標が違う者どうしでも努力量ベースで公平に戦える。ジムCPUは `goal:7000` 固定。

## 5. モンスターの増やし方（第2弾用）

1. `SPR` に16×16ドット絵を追加（pal文字→色。全行16文字、`.`=透過。読み込み時に正規化あり）
2. `SPECIES` に追加: `{name, stage, line, type("P"|"S"|"T"), dex:連番, desc, hybridOnly?}`
3. `PROF` に図鑑詳細を追加: `{hw:"体高/重さ", fl:"フレーバーテキスト"}`（テストが全種の存在を検査する）
4. `MOVES` に技3つ `[名前, 属性, 威力]`（威力目安: 成熟期30〜44）
5. 進化分岐に組み込むなら `evolveAdult` / `evolveChild` を編集
6. 図鑑は `SPECIES`+`PROF` から自動生成されるので追加作業不要。そだてた回数は `G.raised[species]`（孵化・進化時に加算、対戦で見ただけでは増えない）

## 6. テスト（品質保証の再現方法）

- `test-logic.js`: LOGIC区間を抽出して実行。進化全分岐・ゾロ目・コード往復/改ざん・交配・目標歩数スケール・努力ベース同値性など **139項目**（v1.1全パス）
- `smoke.js`: jsdomでUI操作を再現（目標設定→孵化→成熟→対戦決着→交配→ボックス入替→hybrid育成→ジム戦）**17項目**（v1.1全パス）
- `sim.js`: バランス検証（同一目標時、弱い側が最適手・強い側がCPU AI）:
  累計2〜5倍差でも約85%（努力換算の上限=目標30日ぶんで頭打ちのため差が縮む）/ 相性最悪+先手を取られる組合せは0〜6% → 冷凍ボックスの属性違いで選出対策するのが攻略の軸
